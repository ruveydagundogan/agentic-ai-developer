# Day 9 - Pushing Changes

## Summary

- `git push` uploads local commits to the remote.
- The first push for a branch often uses `-u`.
- After upstream is set, plain `git push` is enough.
- Pushing shares your work with others.

## Commands

- `git push` → Uploads commits from the current branch.
- `git push -u origin <branch>` → Sets the branch upstream on first push.

## Notes

- Use `-u` only the first time for a new branch.
- Push often to keep the remote up to date.
- If push fails, pull first or check branch names.
