# Day 6 - Merging

## Summary

- `git merge` brings another branch into the current branch.
- A merge may create a merge commit if histories diverge.
- Conflicts can happen when the same file changed in both branches.
- You can delete a finished branch after it is merged.

## Commands

- `git switch` → Moves to the branch that should receive the merge.
- `git merge <branch>` → Combines changes from another branch.

## Notes

- Always check the branch you are on before merging.
- Resolve conflicts carefully and test after merge.
- Remove old branches when they are no longer needed.
