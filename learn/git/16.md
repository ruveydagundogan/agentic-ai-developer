# Day 16 - Tagging Releases

## Summary

- `git tag` marks a commit as a release point.
- Tags are useful for versioning milestones.
- Annotated tags store author and message details.
- Tags are fixed pointers, unlike branches.

## Commands

- `git tag` → Creates a tag for a commit.
- `git push origin <tag>` → Sends one tag to the remote.
- `git push --tags` → Sends all tags to the remote.

## Notes

- Use annotated tags for official releases.
- Tags are not updated automatically after creation.
- Push tags separately if you want them on the remote.
