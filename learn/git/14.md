# Day 14 - Cherry-Picking

## Summary

- `git cherry-pick` applies one commit onto your current branch.
- It copies a specific change without merging the whole branch.
- Cherry-pick is handy for bug fixes or small updates.
- It can still cause conflicts like a normal merge.

## Commands

- `git cherry-pick <commit>` → Applies one commit from another branch.

## Notes

- Cherry-picking duplicates the selected change.
- Use it when you need one commit but not the full branch.
- Check history after cherry-pick to confirm the result.
