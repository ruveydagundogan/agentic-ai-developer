# Day 12 - Undoing Changes

## Summary

- `git reset` moves the branch pointer and can unstage edits.
- `git revert` creates a new commit that reverses a past commit.
- Use `reset` for local commits and `revert` for shared history.
- `git restore <file>` resets a file to the last commit.

## Commands

- `git reset` → Moves the current branch pointer.
- `git revert` → Makes a new commit that undoes an earlier one.
- `git restore <file>` → Restores a file from the last commit.

## Notes

- `git reset --hard` discards local changes, so use it carefully.
- `git revert` is safer when others already have the commit.
- `git restore` is useful for recovering one file.
