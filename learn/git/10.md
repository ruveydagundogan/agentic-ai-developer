# Day 10 - Pulling Changes

## Summary

- `git pull` downloads remote changes into the current branch.
- It combines fetch and merge in one step.
- Pull before starting new work to stay updated.
- Keeping local branches current reduces conflicts.

## Commands

- `git pull` → Fetches and merges remote updates.

## Notes

- Check `git status` before pulling.
- If conflicts appear, resolve them after the pull.
- Pull regularly when working with others.
