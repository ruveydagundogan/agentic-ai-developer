# Day 13 - Interactive Rebase

## Summary

- `git rebase -i` lets you edit commit history before sharing.
- It can squash, reorder, or change commit messages.
- Interactive rebase is best on local branches only.
- Conflicts can happen and need to be resolved carefully.

## Commands

- `git rebase -i` → Opens interactive rebasing for recent commits.

## Notes

- Do not rewrite history on branches others use.
- Keep commits small and meaningful before rebasing.
- Fix conflicts and continue the rebase until it finishes.
