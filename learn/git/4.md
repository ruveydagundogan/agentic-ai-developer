# Day 4 - Ignoring Files

## Summary

- `.gitignore` tells Git which files to ignore.
- It keeps the repository free of build files and local settings.
- Files already tracked are not ignored by `.gitignore`.
- Add ignore rules early in a project.

## Commands

- `.gitignore` → Lists patterns of files to ignore.
- `git status` → Verifies which files are tracked or ignored.

## Notes

- Ignored files do not appear as untracked.
- If a file is already committed, `.gitignore` does not remove it.
- Common ignored items are logs, build output, and IDE files.
