# Day 11 - Cloning a Repository

## Summary

- `git clone` makes a local copy of an existing repository.
- It downloads all commits and branch history.
- The cloned repo already has an `origin` remote.
- Cloning is the normal way to start on another project.

## Commands

- `git clone` → Creates a local repository from a remote URL.

## Notes

- After clone, you can work immediately without `git init`.
- The `.git` folder is created automatically.
- Clone includes the full history of the source repo.
