# Day 1 - Initializing a Repository

## Summary

- `git init` creates a new local Git repository.
- Git stores history and branches in the hidden `.git` folder.
- A fresh repo has no commits yet.
- `git clone` gives you a full copy of a remote repository.

## Commands

- `git init` → Creates a new Git repository.
- `git clone` → Downloads a repository and its history from a remote.

## Notes

- Do not delete the `.git` folder for an active repo.
- Clone already includes the `.git` directory.
- `git init` only sets up the repo, it does not add files or commits.
