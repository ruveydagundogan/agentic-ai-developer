# Day 3 - Checking Status & History

## Summary

- `git status` shows tracked, modified, and untracked files.
- `git log` shows past commits and the order of changes.
- `git log --oneline` gives a shorter history view.
- `git log --graph` helps see branch structure visually.

## Commands

- `git status` → Shows the current state of the working tree.
- `git log` → Displays the commit history.
- `git log --oneline` → Shows a compact history list.

## Notes

- Use status before editing, committing, or switching branches.
- History helps you find when changes happened.
- Graph view is useful for branch work.
