# Day 2 - Staging & Committing

## Summary

- `git add` moves selected changes into the staging area.
- `git commit` saves staged changes into repository history.
- The staging area lets you control what goes into each commit.
- Good commit messages explain what changed.

## Commands

- `git add` → Stages changes for the next commit.
- `git commit` → Saves staged changes as a new commit.

## Notes

- You can stage only some files and leave others out.
- Commit often to keep history clear.
- Check `git status` before committing.
