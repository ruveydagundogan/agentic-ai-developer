# Day 7 - Resolving Merge Conflicts

## Summary

- Merge conflicts happen when Git cannot merge edits automatically.
- They usually appear when the same file part changed on both branches.
- You fix the file manually and then continue the merge.
- After fixing, stage and commit the result.

## Commands

- `git merge` → Starts the merge that may produce conflicts.
- `git add` → Marks the resolved file as ready.
- `git commit` → Finishes the merge after resolution.

## Notes

- Conflict markers show the conflicting sections.
- Open the file and decide which changes to keep.
- Do not commit until the conflict is fully resolved.
